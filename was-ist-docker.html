Anker lichten - Was ist Docker?

Auf dieser Seite geht behandeln wir das Thema Docker von Anfang an. Es wird erklärt was Docker ist, warum man es nutzen sollte und wer Docker alles nutzt.

Definition

In diesem Abschnitt erfährst du was Docker ist und wozu man es verwenden kann.

Ein Beispiel: Wenn auf einem Mac wird mit einer bestimmten Version von PHP Software entwickelt wird, kann es sein, dass diese auf einem Linux Server nicht gleichermaßen läuft. Selbst wenn beide Systeme mit derselben Version von PHP arbeiten, unterscheiden sich beispielsweise Konfigurationen. Dies kann Probleme verursachen und die Behebung dieser Barrieren eine Menge Arbeitsaufwand bedeuten. Docker wird zunehmend populär, weil es für diese Problematik eine Lösung findet.


Docker ist ein Open Source-Projekt zur Automatisierung der Bereitstellung von Anwendungen, die in der Cloud oder lokal ausgeführt werden können. Dazu nutzt Docker sogenannte Container und grenzt sich so auch gegenüber virtuellen Maschinen ab. Docker-Container enthalten eine Software mit all ihren Abhängigkeiten. Dadurch braucht ein Container weniger Ressourcen als virtuelle Maschinen und ermöglicht das plattformunabhängige Arbeiten. Docker sorgt dafür, dass sich die entwickelte Software auf jedem Gerät exakt gleich verhält. Diese und viele weitere nützliche Eigenschaften, tragen zu dem Erfolg von Docker bei und sorgen dafür, dass die Nutzeranzahl über die letzten Jahre stetig gestiegen ist - Tendenz steigend.

Warum Docker?

Hier erfährst du wozu man Docker verwenden sollte und wo die Vor- und Nachteile der Containerisierungstechnologie liegen. Es soll auch klar gemacht werden, dass Docker sich grundsätzlich von Virtuellen Maschinen unterscheidet.

Vorteile

Container-Technik

Docker stellt für die jeweiligen Container und Images immer die passende Entwicklungsumgebung her. So werden Frameworks und Libraries geladen, aber unnötige Elemente ignoriert. Durch die Benutzung des Host OS-Kernels fällt auch der Hypervisor weg und wird durch die Docker Engine ersetzt. So können Ressourcen gespart werden, da der Server besser genutzt wird.

Docker vs Virtuelle Maschine

Speicherplatz

Container sind in der Regel nicht größer als 100 MB. Virtuelle Maschinen benötigen durch das eigene Betriebssystem mehrere Gigabyte. So kann ein Server viel mehr Container hosten als VM. Dadurch werden Ressourcen gespart.

Boot-Zeit

Da die Container kein eigenes OS besitzen und recht klein sind benötigen sie nur wenige Augenblicke zum Start, während VM einige Minuten brauchen, bis sie betriebsbereit sind. Dadurch könnte man theoretisch Container beenden um Strom zu sparen und bei Bedarf wieder schnell booten.

Modularisierung

Statt eine komplette und komplexe Applikation in einem Container zu betreiben arbeiten verschiedene Modul-Container zusammen. Dieser sogenannte Mircoservice-Ansatz ermöglicht die Aufteilung von bspw. einer Datenbank. So können auch einfach Komponenten geändert oder ausgetauscht werden ohne die gesamte Applikation umzuschreiben.

Versionierung

Mit Docker können verschiedene Versionen eines Containers erstellt, untersucht und wiederhergestellt werden. Außerdem werden Komponenten aus den vorhergehenden Schichten wiederverwendet, umso den Container schlank zu halten.

Nachteile

“Hauptsache Container” (Hype)

Anstatt die angesprochene Modularisierung von Docker zu nutzen wird eine gesamte Applikation in einen Container gepackt. In diesem Fall könnte man genauso eine VM verwenden.

GUI

Docker ist für Server konzipiert. Deshalb ist die Unterstützung für Applikationen mit einer GUI wie bspw. X11 video forwarding sehr schlecht. Anwendungen wie Photoshop können nicht mit Docker umgesetzt werden.

Root Rechte

Die Engine von Docker, der Docker Daemon, benötigt für den Einsatz root Rechte. Das ist im Prinzip das gleiche wie Administratorrechte auf einem Windows Systemen. Da der Daemon als root läuft, hat jeder Benutzer, der Container starten kann, im Prinzip root-Rechte auf dem System. So kann auf Verzeichnisse zugegriffen werden, in dem bspw. Passwörter gespeichert werden.

Sicherheit

Da Docker eine Entwicklungsumgebung für Software bereitstellt sollte man denken, dass alle Risiken behoben sind, jedoch darf der Sicherheitsaspekt nicht vernachlässigt werden. Da Anwendungen direkt auf den Host OS-Kernel zugreifen muss können leichter Sicherheitslücken auftreten als bspw. bei einer VM. Deshalb ist ein gewisses Grundvertrauen in die Entwickler einer Applikation nötig. Docker ist außerdem keine Wunderwaffe für die Sicherheit des Gesamtsystems, sondern nur eine zusätzliche Ebene.

Docker ersetzt nicht VMs

Auch wenn Docker immer populärer wird werden Virtuelle Maschinen nicht gleich in großem Maß aussterben. Experten sehen die Zukunft von zwei komplementären Technologien. Docker ist leichter und agiler, als Anwendungen auf VMs. Lorenzo Fontana drückt es so aus: „Container sind Prozesse, VMs sind Server.“

Wer nutzt Docker?

Dieser Abschnitt zeigt, wie populär Docker ist und welche bekannte Unternehmen Docker in ihrem Alltag nutzen.

Docker ist ein Erfolgsmodell. Dafür sprechen auch die offiziellen Daten der Website des Unternehmens. Über 80 Milliarden Container wurden bisher heruntergeladen. Des Weiteren findet man über 2 Millionen Anwendungen in Docker Hub. Doch nicht nur die über 100.000 Third-Party-Projekte nutzen die Container Software, Docker kann auch über 650 kommerzielle Kunden vorweisen. Dazu zählen auch namhafte Kunden wie beispielsweise PayPal, Visa oder GSK.


Die Popularität von Docker lässt sich am besten mit dem folgenden Diagramm zeigen. Man sieht das Interesse von Docker und der beliebtesten virtuellen Maschine (Virtual Box) im Verlauf der letzten Jahre. Docker, das erst 2013 veröffentlicht wurde, schaffte es innerhalb von nur drei Jahren das Interesse auf das gleiche Level anzuheben wie virtuelle Maschinen.


Ein wichtiger Faktor dafür ist auch die Docker-Community. Mit 32.000 Sternen auf GitHub und über 200 Meetups hat Docker eine solide Basis an Beitragenden. Jährlich findet auch aus diesem Grund eine DockerCon statt, auf der Neuerungen vorgestellt werden und Workshops stattfinden. Bei Docker handelt es also längst nicht mehr um ein Nischenprodukt, sondern um eine Lösung, die lange Zeit gesucht wurde. Das zeigt auch die Zahl der Docker Job Angebote, die seit Veröffentlichung um 77.000% gestiegen ist.
